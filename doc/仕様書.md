# FIRE Countdown アプリ仕様書

## 1. アプリ概要

### アプリ名

FIRE Countdown

### コンセプト

投資額や将来資産額そのものではなく、**経済的自由までの残り時間**を可視化するアプリ。

通常の投資シミュレーターが「将来いくらになるか」を示すのに対し、本アプリは「いつFIREできるか」「今の追加投資で何日FIREが近づくか」を示す。

### 最大の特徴

**お金を時間に変換すること。**

例：

```text
今1万円を追加投資すると
FIRE達成が2日早まります
```

---

## 2. MVP方針

MVPでは、実績管理は行わない。

本アプリはあくまで、入力された前提条件に基づくシミュレーションアプリとする。

### MVPで扱うもの

* 現在資産額
* 毎月追加投資額
* 想定年利
* 目標資産額
* 達成予定日
* 残り期間
* 1万円追加投資による短縮日数
* 元本と運用益の積み上げグラフ
* 楽観・標準・悲観シナリオ

### MVPで扱わないもの

* 実績入力
* 進捗率
* 通知
* 支出によるFIRE延長表示
* 税金考慮
* インフレ考慮
* 口座連携
* Coast FIRE
* Side FIRE
* 暴落シナリオ

---

## 3. 入力項目

### 必須入力

| 項目       | 内容         |            例 |
| -------- | ---------- | -----------: |
| 現在資産額    | 現在の投資可能資産  |  30,000,000円 |
| 毎月追加投資額  | 毎月積み立てる金額  |     300,000円 |
| 目標資産額    | FIRE達成ライン  | 100,000,000円 |
| 標準シナリオ年利 | 基準となる想定利回り |           7% |

### 任意入力

| 項目       | 内容              |   例 |
| -------- | --------------- | --: |
| 楽観シナリオ年利 | 良好な市場環境を想定した利回り | 10% |
| 悲観シナリオ年利 | 厳しい市場環境を想定した利回り |  3% |

楽観・悲観シナリオはオプション扱いとする。
未入力の場合は標準シナリオのみ表示する。

---

## 4. 出力項目

### 4.1 達成予定日

標準シナリオをもとに、目標資産額へ到達する予定日を表示する。

```text
達成予定日
2034年2月15日
```

### 4.2 残り期間

現在日から達成予定日までの期間を表示する。

```text
残り
7年8か月3日
```

### 4.3 1万円短縮効果

現在資産に1万円を追加した場合、FIRE達成予定日が何日早まるかを表示する。

```text
今1万円を追加投資すると
FIRE達成が2日早まります
```

### 4.4 達成時の内訳

標準シナリオの達成時点における以下を表示する。

| 項目    | 内容            |
| ----- | ------------- |
| 達成時資産 | 目標到達時点の資産額    |
| 累計元本  | 現在資産 + 追加投資累計 |
| 運用益   | 達成時資産 − 累計元本  |

## 5. シナリオ機能

### 5.1 標準シナリオ

必須の基準シナリオ。
メイン表示、1万円短縮効果、元本・運用益グラフは標準シナリオを基準にする。

### 5.2 楽観シナリオ

任意入力。
標準より高い利回りを想定した場合の達成予定日を表示する。

### 5.3 悲観シナリオ

任意入力。
標準より低い利回りを想定した場合の達成予定日を表示する。

### 表示例

| シナリオ |  年利 | 達成予定日    | 残り期間   |
| ---- | --: | -------- | ------ |
| 楽観   | 10% | 2032年3月  | 5年9か月  |
| 標準   |  7% | 2034年2月  | 7年8か月  |
| 悲観   |  3% | 2038年11月 | 12年5か月 |

## 6. グラフ仕様

### 6.1 元本・運用益の積み上げ棒グラフ

標準シナリオにおいて、年ごとの資産推移を表示する。

```text
総資産 = 元本 + 運用益
```

表示項目：

* 元本
* 運用益

目的：

* 追加投資による積み上げ
* 複利による運用益の増加
* 後半ほど運用益が大きくなる構造

を直感的に見せる。

### 6.2 シナリオ別資産推移グラフ

楽観・悲観シナリオが入力されている場合、標準シナリオと合わせて資産推移を比較表示する。

表示対象：

* 楽観
* 標準
* 悲観
* 目標資産ライン

## 7. 計算仕様

### 7.1 月次複利

資産推移は月次複利で計算する。

```text
月利 = 年利 / 12
```

毎月以下を繰り返す。

```text
資産額 = 資産額 × (1 + 月利) + 毎月追加投資額
元本 = 元本 + 毎月追加投資額
```

資産額が目標資産額以上になった時点で達成とする。

### 7.2 達成日数

到達月を判定したうえで、到達月の前月資産額から到達月資産額までを線形補間し、概算日数を算出する。

```text
月内進捗率 = (目標資産額 - 前月資産額) / (到達月資産額 - 前月資産額)
達成日数 = (到達月数 - 1 + 月内進捗率) × 30.4375
```

月内進捗率は0から1の範囲に丸める。

### 7.3 達成予定日

```text
達成予定日 = 今日 + 達成日数
```

### 7.4 1万円短縮効果

以下の2パターンを比較する。

```text
A: 現在資産のままシミュレーション
B: 現在資産 + 10,000円でシミュレーション
```

```text
短縮日数 = Aの達成日数 - Bの達成日数
```

## 10. 注意事項

### 金融助言ではない

本アプリは投資判断を助言するものではなく、入力された前提条件に基づく簡易シミュレーションを提供するものである。

### シミュレーションは将来を保証しない

利回りは一定ではなく、実際の市場環境によって達成予定日は大きく変動する。

### 表示文言の注意

「FIREまであと○日」と断定しすぎず、以下のように表現する。

```text
現在の想定では
2034年2月15日に到達予定
```

## 12. アプリの本質

FIRE Countdownは、単なる投資シミュレーターではない。

本質は、

```text
お金を人生の時間に翻訳すること
```

である。

ユーザーはこのアプリを通じて、

```text
追加投資額
利回り
目標資産
```

ではなく、

```text
経済的自由までの残り時間
```

としてFIRE計画を理解できる。
